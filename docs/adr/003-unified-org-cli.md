# ADR-003: Unified Org-Scoped CLI Entry Point

- **Status:** Accepted
- **Date:** 2026-06-28
- **Spec:** [0006](../specs/0006-unified-cli-and-tracing.md), [0012](../specs/0012-synthetic-org-divisions-and-handoffs.md)

## Context

Blue Hen RE runs multiple product orgs (hub, research-rag, benchmark-lab, dumbmodel) under one
monorepo. Each org has its own workspace API key and Vercel project, but agents and humans need
the **same command surface** for lifecycle actions and division handoffs.

Without org scoping, handoffs require copying env vars or guessing which workspace key is active.

## Decision

1. **`synth` is the single entry point** for all orgs — humans, CI, Eve, and Cursor agents.
2. **Org scope** via global `--org <siteId>` or `SYNTH_ORG`; credentials resolve from
   `data/workspaces/{siteId}.env` (generated by `bootstrap_orgs.py`).
3. **`@synthaembed/fleet`** owns workspace credential resolution (`loadWorkspaceCredentials`).
4. **`synth org handoff`** records division handoffs to the operations ledger (Spec 0012).
5. **Eve parity:** `synthFor(actor, siteId?)` in `apps/synthorg/agent/lib/synth.ts` uses the
   same resolver.

## Consequences

- Agents can hand off with one line: `synth --org research-rag org handoff --from research --to bd --note "..."`.
- Product teams keep owning their site + workspace; platform owns the CLI contract.
- Bootstrap must run per environment so workspace env files exist locally and in CI.

## Alternatives considered

| Option | Rejected because |
|--------|------------------|
| Per-site CLIs (`hub-cli`, `arxiv-cli`) | Fragments tracing; agents need N entry points |
| Env-only (no `--org` flag) | Error-prone in multi-org sessions |
| Direct core-api calls from agents | Breaks Spec 0006 uniform access layer |

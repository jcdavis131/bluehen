## Findings

### research-rag
- 🟢 Real, working Phase A site. Routes: `/`, `/research-lab`, `/feedback`, plus 4 BFF routes that thin-reexport `@synthaembed/ui-fleet/routes` — `app/api/search/route.ts:1`, `app/api/status/route.ts:1`, `app/api/models/route.ts:1`, `app/api/feedback/route.ts:1`. Home + feedback pages use `PageHeader`/`SiteSubnav` from the design system — `app/page.tsx:3`, `app/feedback/page.tsx:1`.
- 🟢 Spec 0015 consent gate respected. `venture.dataConsent` declares "Query logging deferred until consent UI ships (Spec 0015 gate)" (`config/fleet.json:232`). The shared `POST_search` handler (`packages/ui-fleet/src/routes/index.ts:19-36`) only forwards `query/k/truncateDims/quant` to `core-api /v1/search` — no query persistence at the BFF layer. No consent UI is shipped, and no logging is wired, so the gate is honoured.
- 🟡 Convention drift on `/research-lab`. `app/research-lab/page.tsx:51-210` uses the older `fleet-card` / `fleet-grid` / `fleet-badge` classes plus extensive inline `style={{...}}` for layout, accent colours, and stage tints. The home page (`app/page.tsx:32-47`) uses the newer `bh-card` / `bh-btn` / `bh-badge` design-system primitives. The two pages read like different vintages.
- 🟡 `/research-lab` omits `SiteSubnav`. Every other top-level page (`app/page.tsx:29`, `app/feedback/page.tsx:19`) renders `<SiteSubnav items={nav} currentPath=... />`. The research-lab page does not, so users landing on it lose site navigation — `app/research-lab/page.tsx:55-56`.
- 🟡 `ArxivExamDemo` mixes class systems: `fleet-card`/`fleet-badge` for the status panel (`components/ArxivExamDemo.tsx:33-43`) and `bh-textarea`/`bh-btn` for the form (`components/TierComparePanel.tsx:111-138`). Pick one vocabulary per site.
- 🟢 `metadata` is set on every route (`app/page.tsx:6`, `app/research-lab/page.tsx:3`, `app/feedback/page.tsx:4`). Layout pulls in `@synthaembed/ui-fleet/styles.css` and wraps `FleetShell siteId="research-rag"` (`app/layout.tsx:1-12`). `next.config.mjs` transpiles both workspace packages.
- 🟢 API wiring is consistent: no hardcoded `localhost:8000`; the BFF reads `SYNTH_API_BASE_URL ?? NEXT_PUBLIC_API_BASE_URL ?? http://localhost:8000` server-side and gates on `SYNTH_API_KEY` (`packages/ui-fleet/src/site-api.ts:26-37`). The site itself never sees the key.

### benchmark-lab
- 🟢 Most complete of the four. Routes: `/`, `/queue`, `/try`, `/certify`, `/feedback`, plus 5 BFF routes. Uses `@synthaembed/eval-public` for `BENCHMARK_EXAMS`, `RAG_TIERS`, `hallOfCone` (`app/page.tsx:2`) — the only site pulling from the eval package, which is correct for a Validation Lab.
- 🟢 Strong design-system adherence: every page uses `PageHeader` + `SiteSubnav` + `bh-card` / `bh-grid` / `bh-table` / `bh-btn` primitives (`app/page.tsx:17-92`, `app/certify/page.tsx:30-69`, `app/queue/page.tsx:52-96`, `app/try/page.tsx:15-55`). `ProgressMeter` from `ui-fleet` used for the leaderboard (`app/page.tsx:78-86`).
- 🟢 Spec 0012 BD-promotion queue is wired end-to-end: `app/queue/page.tsx` fetches `/api/queue` → `GET_bdQueue` → `siteBdQueue()` → `core-api /v1/bd/queue`. The static fallback `data/bd-queue.json` matches the runtime shape (id, siteId, method, status, recipe, gates, evidenceRef). `GLOSSARY.bdQueue` is used for the title.
- 🟡 Stale phase copy. `app/try/page.tsx:33` says "YAML exam runner ships in Phase 3." The site is declared Phase A in `config/fleet.json:162`. "Phase 3" is undefined in the current phase taxonomy (A / B / all) — should read "Phase B" or "post-Phase A".
- 🟢 `venture.dataConsent` ("Customer eval sets under NDA; published scorecards only with clearance", `config/fleet.json:192`) is consistent with the surface: `LiveSearchPanel` only forwards queries through the shared `POST_search` (no upload, no persistence). `/certify` describes NDA + published scorecard workflow but collects no data itself (`app/certify/page.tsx`).
- 🟢 `metadata` set on all pages; layout matches the fleet pattern (`app/layout.tsx:1-12`). `.gitignore` covers `.vercel` (matches siblings).

### finance-lab
- 🟢 Phase B guardrail held — no live trading logic. The only API route is `/api/waitlist` (`app/api/waitlist/route.ts:13-50`); it appends `{email, interest, receivedAt, source}` to `data/leads/waitlist.jsonl`. No order placement, no broker integration, no real-capital code path. `app/simulate/[platform]/page.tsx:22` badge explicitly reads "simulation only" and the lead copy states "Not live trading." `scripts/omni_simulate.py` and `scripts/omni_loop.py` exist at the repo root, matching the CLI snippets shown on the page.
- 🟢 Consent gate consistent. `venture.dataConsent` = "Waitlist email stored for this list only (data/leads/waitlist.jsonl)" (`config/fleet.json:295`). The form copy says "your email is stored for this list only — see the privacy note" with a link to `bhenre.com/legal/privacy` (`app/page.tsx:30-32`). The route only persists email + interest + timestamp — no extras, no third-party forwarding unless `CONTACT_WEBHOOK_URL` is set (`app/api/waitlist/route.ts:37-48`). Root `.gitignore` excludes `/data/` so the JSONL never commits.
- 🟡 Stale external spec link. `app/page.tsx:42` links to `https://github.com/jcdavis131/henington-homes/blob/main/specs/0013-omni-market-alpha-engine.md` — that's the legacy repo. Spec 0013 actually lives in this monorepo at `specs/0013-omni-market-alpha-engine.md`. Link should be relative (`/specs/0013-...` is not web-served; either drop the link or point to the in-repo path on the current GitHub remote).
- 🟡 Missing `.gitignore`. research-rag, benchmark-lab, and training-console all ship a site-level `.gitignore`; finance-lab does not. Root `/data/` rule covers the JSONL, but `.next/` and `.vercel/` for this site rely on global ignores only — inconsistent with siblings.
- 🟡 No `metadata` export on `app/page.tsx` or `app/simulate/[platform]/page.tsx`. The layout sets a site-wide title/description (`app/layout.tsx:4-7`), so the home page is covered, but the dynamic `/simulate/[platform]` page renders with the generic site title rather than e.g. "Kalshi paper trade — Simulation Lab". Minor SEO/UX gap.
- 🟢 Layout + `FleetShell siteId="finance-lab"` consistent with siblings (`app/layout.tsx:1-17`). Uses `bh-card` / `bh-btn` / `bh-link` / `bh-label` / `bh-input` / `bh-select` design-system primitives throughout (`app/page.tsx`, `components/WaitlistForm.tsx:39-62`). `WaitlistForm` has proper `role="alert"` / `role="status"` ARIA (`components/WaitlistForm.tsx:32,58`).

### training-console
- 🔴 Dead route link — `/runs/[id]` does not exist. `app/page.tsx:85` renders `<Link href={`/runs/${encodeURIComponent(run.id)}`}>` for every run card, but there is no `app/runs/[id]/page.tsx` (Glob `apps/sites/training-console/app/runs/**` → 0 files). Clicking any run 404s. `components/RunDetail.tsx` is fully implemented (manifest + metric polling + chart grouping + collapse alert + config table, `RunDetail.tsx:17-177`) but is never imported or mounted anywhere — orphaned component. This is the highest-impact gap in the lane.
- 🟡 Bypasses the fleet BFF pattern. `lib/api.ts:5-17` reads `NEXT_PUBLIC_TELEMETRY_URL ?? http://localhost:8100` and `NEXT_PUBLIC_TELEMETRY_KEY` and fetches `/v1/runs`, `/v1/runs/{id}/metrics`, `/v1/runs/{id}/events` directly from the **client** (page.tsx and RunDetail.tsx are `"use client"`). Every other site routes through `app/api/*` BFF handlers that keep `SYNTH_API_KEY` server-side. Because `NEXT_PUBLIC_*` vars are inlined into the bundle, any tenant key shipped here is exposed to the browser. Acceptable only while the site has no public domain (`config/fleet.json:238` domain is null); becomes a 🔴 the moment it deploys publicly.
- 🟡 Design-system divergence. `app/console.css` defines a parallel token set (`--page`, `--surface-1`, `--ink`, `--s1..--s5`, `--status-*`) and a parallel class vocabulary (`console-*`, `run-card`, `chart-panel`, `stat-tile`, `status-pill`, `alert-banner`). The layout still imports `@synthaembed/ui-fleet/styles.css` and wraps `FleetShell` (`app/layout.tsx:1-21`), so the fleet shell chrome renders with `bh-*` tokens while the page body renders with `console-*` tokens. Intentional for an internal dark observatory, but it is the only site in the lane not using `PageHeader` / `SiteSubnav` / `bh-card` primitives — no `getSiteCircuit` / `getSiteNav` call either.
- 🟢 Observability scope is correct. `config/fleet.json:236-261` declares training-console with `role: training-observability`, no `venture` block, no domain — matches Spec 0015 ("Internal, not ventures: training-console — R&D observatory"). No monetisation, no CTA, no lead capture — the page is purely a run telemetry viewer. Good lane discipline.
- 🟢 Telemetry contract is documented and structured. `lib/types.ts` defines `RunManifest`, `MetricRow`, `EventRow`, `Series`, `ChartEvent`. `lib/api.ts` uses `cache: "no-store"` and `Authorization: Bearer ${KEY}` when a key is present. `RunDetail.tsx` implements incremental polling (`seen.current` / `seenEvents.current` cursors), stops polling when `status !== "running"` (`RunDetail.tsx:47-50`), and surfaces a stale-data banner with `role="status"`.
- 🟢 Accessibility fallback built in. `LineChart.tsx:210-243` renders a `<details>` data table keyed by step (union across series), so unaligned series never get values attributed to the wrong step. SVG charts carry `role="img"` + `aria-label` (`LineChart.tsx:106-107`). Status pills use dot + label, never colour alone (`app/console.css:73-84`).
- 🟡 `.gitignore` covers `.next/`, `node_modules/`, `.vercel` (`app/.gitignore:1-3`) — slightly broader than siblings (research-rag and benchmark-lab only list `.vercel`), which is fine but inconsistent.

## Risk
- **training-console `/runs/[id]` 404 (🔴, high)** — the entire run-detail view is unreachable. `RunDetail.tsx` and `LineChart.tsx` are dead code until the route is added. Any operator clicking a run card hits a 404 and loses trust in the observatory surface. Cheap fix: one dynamic route file mounting `<RunDetail runId={params.id} />`.
- **training-console client-exposed telemetry key (🟡, medium)** — fine while internal, becomes a credential leak the day training-console gets a public domain or is deployed to Vercel without reconsidering the env var prefix. Should move to a BFF route (`app/api/runs/...`) with a server-only `TELEMETRY_KEY` before any public deploy.
- **research-rag `/research-lab` convention drift (🟡, low)** — visual inconsistency between sibling pages on the same site; not a correctness risk but signals the design-system migration is half-done. The `fleet-*` classes may not even exist in the current `ui-fleet` stylesheet (the home page uses `bh-*`), so the research-lab cards may be unstyled or rely on legacy CSS.
- **finance-lab stale spec link (🟡, low)** — points to the legacy `henington-homes` repo; readers leave the monorepo. Easy fix.
- **benchmark-lab "Phase 3" copy (🟡, low)** — undefined phase in the current taxonomy; reader confusion only.
- **finance-lab missing `.gitignore` (🟡, low)** — relies on root ignores; risk of accidental commit if a site-local build artifact ever lands outside `/data/` and `.next/`.

## Recommendation
1. **Add `apps/sites/training-console/app/runs/[id]/page.tsx`** mounting `<RunDetail runId={(await params).id} />` (one file, ~10 lines). Verify with `pnpm --filter @synthaembed/training-console dev` and clicking a run after `uv run python -m runboard demo && uv run python -m runboard serve`. Highest ROI fix in the lane.
2. **Move training-console telemetry behind a BFF.** Add `app/api/runs/route.ts` + `app/api/runs/[id]/{metrics,events}/route.ts` that read a server-only `TELEMETRY_URL` / `TELEMETRY_KEY` (drop the `NEXT_PUBLIC_` prefix). Update `lib/api.ts` to call `/api/runs/...`. Required before any public deploy of this site.
3. **Migrate `app/research-lab/page.tsx` to `bh-*` primitives** and add `<SiteSubnav items={getSiteNav("research-rag")} currentPath="/research-lab" />`. Wrap with `PageHeader` for eyebrow/title consistency. Same migration for `ArxivExamDemo` / `TierComparePanel` (`fleet-card` → `bh-card`, `fleet-badge` → `bh-badge`, `fleet-grid` → `bh-grid`).
4. **Fix finance-lab spec link** to point at the in-repo `specs/0013-omni-market-alpha-engine.md` (or remove the hyperlink and keep "Spec 0013"). Add a site-level `.gitignore` matching siblings. Add `metadata` exports to `app/page.tsx` and `app/simulate/[platform]/page.tsx` (dynamic title per platform).
5. **Update benchmark-lab `/try` copy** from "Phase 3" to "Phase B" (or "post-Phase A") to align with the current phase taxonomy in `config/fleet.json`.
6. **Optional hygiene pass**: confirm `fleet-card` / `fleet-grid` / `fleet-badge` classes are actually defined in `@synthaembed/ui-fleet/styles.css` (or legacy CSS); if not, the research-lab page is rendering unstyled today and recommendation 3 becomes a 🔢 bug fix, not a polish item.

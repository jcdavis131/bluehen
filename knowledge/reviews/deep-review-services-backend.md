## Findings

- 🔴 **`governance.get_trace` does not exist — `GET /v1/trace/{trace_id}` is broken.** `main.py:77-79` calls `governance.get_trace(trace_id)`, but `governance.py` has no such function. Lines `governance.py:142-163` are a function body with no `def` line — dead code after `site_id_for_workspace`'s `return` at `:139`. Verified: `python -c "from app.services import governance; print(hasattr(governance, 'get_trace'))"` → `False`. Calling the route raises `AttributeError` → 500. This is the same symptom referenced by the pre-existing SEC-003 note in `knowledge/reviews/security.md:41,58-60`; the function shape was lost (fail-closed by accident), and the dead body still opens `db_session()` with no `workspace_id`, so a naive `def get_trace(trace_id):` restore would re-introduce the cross-tenant RLS bypass. — `services/core-api/app/main.py:77-79`, `services/core-api/app/services/governance.py:131-163`
- 🔴 **Trainer tests are red against the skeleton (3 failed, 8 errors).** `services/trainer/tests/test_modal_app.py` expects `_post_callback`, real checkpoint files on the volume, `result["pairs"]`, and float `ndcg10`/`effectiveRank` from `evaluate`. `services/trainer/modal_app.py` is a documented skeleton ("TODO: real training loop", "TODO: persist checkpoint") with no callback helper, `domain_adapt` returns `{"stage","status","baseModel"}` (no `pairs`), `evaluate` returns `None` metrics, and `compress_and_register` writes nothing. The two files have diverged; either implement the body or skip/xfail the suite. — `services/trainer/modal_app.py:31-84`, `services/trainer/tests/test_modal_app.py:88-267`
- 🟡 **Worker deploy is gated by charter, not by eval-harness pass.** `services/worker/main.py:122-130` calls `deploy_model(...)` whenever `handoffs.charter_allows_deploy(site_id, model_version)` is true — and logs `"charter-approved (gates pending)"` when `gates_passed` is False. Eval gates (spec 0008) are recorded in the ledger but do not block the worker's auto-deploy. If a charter is broad (`modelVersion: "*"`), a model that failed gates still gets deployed + indexed. Either require `gates_passed` for the auto path, or document that charter is the sole deploy gate and eval gates are advisory. — `services/worker/main.py:81-130`
- 🟡 **Worker hardcodes serving tier on auto-deploy.** `deploy_model(..., truncate_dims=256, quant="int8", ...)` ignores the recipe's `asn.matryoshkaDims` and any per-site charter tier. — `services/worker/main.py:127-128`
- 🟡 **`seed.ts` is not idempotent.** The header admits "re-running creates duplicates" (`seed.ts:7-8`). `createSalesChannelsWorkflow` / `createRegionsWorkflow` / `createProductsWorkflow` / `createApiKeysWorkflow` run unconditionally — no handle/lookup guard. The sibling `add-certification.ts` does the right thing (`query.graph` for existing handle → skip). Bring `seed.ts` up to the same idempotent-seed-script standard (look up by name/handle before insert, or wrap in a `WHERE NOT EXISTS` guard). — `services/commerce/src/scripts/seed.ts:20-92`
- 🟡 **`document_chunks` table is not in SQLAlchemy metadata.** It exists only in `alembic/versions/002_pgvector_chunks.py` (raw SQL) and is read/written via raw `text()` in `indexing.py` and `search.py`. `ensure_schema()` → `Base.metadata.create_all` (`database.py:52-61`) therefore creates the ORM tables **without** `document_chunks` and **without** any RLS policies (policies live only in Alembic). A process that bootstraps via `ensure_schema` (e.g., dev `USE_MEMORY=0` first run without `pnpm db:migrate`) will 500 on `/v1/search` / `/v1/admin/deploy` and runs with no RLS. `main.py:30-33` swallows the `ensure_schema` exception, hiding a failed migrate. This is SEC-009, still open. — `services/core-api/app/database.py:52-61`, `services/core-api/app/main.py:25-33`, `services/core-api/alembic/versions/002_pgvector_chunks.py`
- 🟡 **`require_tenant` admin impersonation is unvalidated (SEC-008).** `auth.py:54-58` trusts `x-synth-workspace-id` as a raw `UUID(wid)` with no existence check and no try/except — a malformed UUID raises `ValueError` → 500, and an admin key holder can assume any workspace identity. Pre-existing; pair with the shared-static-admin-key concern below. — `services/core-api/app/auth.py:47-69`
- 🟡 **Admin auth is a single shared static bearer (SEC-004, partially mitigated).** `config.py:20-21` adds the production fail-fast (`API_SECRET_KEY == "change-me-32-bytes-min"` raises when `ENVIRONMENT=production`) — good, partial fix. But there is still one secret for every admin surface + worker + CLI, no per-admin keys, no rotation story. — `services/core-api/app/config.py:14-21`, `services/core-api/app/auth.py:35-44`
- 🟡 **Telemetry routes are admin-gated but the filesystem stores are still global (SEC-001/002).** `main.py:387-441` correctly wraps `/v1/runs*` and `/v1/datalab/datasets` in `require_admin` with an explicit comment citing SEC-001/002 — route exposure is closed. The underlying stores (`data/runs`, `data/datalab`) have no `workspace_id` concept, so the moment these open to workspace keys (per the comment's "tenant-scoped stores need a spec"), the leak returns. Latent, not active. — `services/core-api/app/main.py:387-441`, `services/core-api/app/services/telemetry.py`
- 🟡 **Corpus ingestion accepts absolute paths / `..` traversal (SEC-005).** `data._load_corpus_docs` does `Path(corpus_uri)` and uses it verbatim when `is_absolute()`; relative paths are re-based under `CORPORA_DIR/<site_id>` only when they *exist* there, otherwise fall back to `CORPORA_DIR/<site_id>/corpus.jsonl` — but a crafted `../` relative path that resolves to an existing file outside `CORPORA_DIR` is read and ingested into the caller's collection, then retrievable via `/v1/search`. Pre-existing; `runboard/store.py` and `datalab/okf.py` already implement the correct resolve-and-verify-parent guard — mirror it here. — `services/core-api/app/services/data.py:18-43`
- 🟡 **`record_ledger` lets a tenant burn their own ceiling with arbitrary `costUsd`.** `post /v1/ledger` (`main.py:87-89`) is tenant-authenticated and passes `entry` straight through; `governance.record_ledger` (`governance.py:60-85`) reads `cost = float(entry.get("costUsd", 0))` and decrements `spent_usd_today` via an admin session. Not cross-tenant (workspace_id comes from `TenantCtx`), but a tenant can self-DoS or inflate spend reporting. Validate `costUsd` server-side or restrict it to worker/admin paths. — `services/core-api/app/main.py:87-89`, `services/core-api/app/services/governance.py:60-85`
- 🟡 **Worker has no `pyproject.toml` and is not a uv workspace member.** `services/worker/main.py:11-12` does `sys.path.insert(0, ".../core-api")` to import `app.*`. Root `pyproject.toml:9-17` lists `services/core-api` and `services/trainer` as members but not `services/worker`. The Dockerfiles copy and run it directly, so it works in deploy — but `uv run python services/worker/main.py` from a fresh clone relies on core-api's transitive deps. Add a minimal `pyproject.toml` (or make it a workspace member) and drop the `sys.path` hack. — `services/worker/main.py:11-12`, `pyproject.toml:8-17`
- 🟡 **Trainer declares heavy ML deps as core, not extras.** `services/trainer/pyproject.toml:6-13` puts `torch>=2.2`, `transformers>=4.44`, `tokenizers>=0.19`, `modal>=0.64` in core deps. Trainer is a workspace member (`pyproject.toml:16`), so `uv sync --all-packages` pulls torch for everyone who syncs the workspace. Per dependency-hygiene, move these to `[project.optional-dependencies]` (`model`, `gpu`) so non-trainer paths don't install them. — `services/trainer/pyproject.toml:6-13`
- 🟡 **`asn_engine` is imported but not declared by core-api.** `models_svc.embed_texts` does `from asn_engine.model import ASNEncoder` inside the function (`models_svc.py:160`), and `eval.py` imports `eval_harness.runner` inside the function. Neither `asn-engine` nor `eval-harness` is in `core-api/pyproject.toml` deps (only `omni-sim`, `runboard`, `datalab`). It works only because `uv sync --all-packages` installs all workspace members. If core-api is ever installed standalone (e.g., a slim deploy image), `embed`/`eval`/`deploy` will `ImportError` at call time. Declare them (or mark as optional). — `services/core-api/pyproject.toml:6-13`, `services/core-api/app/services/models_svc.py:160`, `services/core-api/app/services/eval.py:47`
- 🟢 **Migrations 001→006 form a coherent RLS sequence.** 001 creates tenant tables + `ENABLE ROW LEVEL SECURITY` + `tenant_isolation` policy; 002 adds `document_chunks` with RLS; 003 creates `synthaembed_tenant` role + grants; 004 adds `tenant_read_self` on `corporate_workspaces`; 005 grants sequence usage (so tenant INSERT works); 006 grants `UPDATE (spent_usd_today)` for budget burn. `down_revision` chain is intact. — `services/core-api/alembic/versions/001..006`
- 🟢 **`db_session` correctly threads RLS.** `database.py:32-49` sets `app.workspace_id` GUC + `SET LOCAL ROLE synthaembed_tenant` only when `workspace_id` is not None, and only when `not USE_MEMORY`. Tenant-scoped services (`data`, `jobs`, `search`, `indexing`, `models_svc`, `eval`, `governance.record_span/list_ledger/get_budget`) all pass `workspace_id`. — `services/core-api/app/database.py:32-49`
- 🟢 **Commerce is provider-agnostic with a fail-fast default.** `medusa-config.ts:26-44` wires Stripe only when `STRIPE_API_KEY` is set; otherwise `modules: []` (no payment provider). `:7-12` throws in production when `JWT_SECRET`/`COOKIE_SECRET`/`DATABASE_URL` are missing. Matches `abstract-the-provider` (Medusa is the default; Stripe is opt-in) and SEC-004's fail-fast pattern. — `services/commerce/medusa-config.ts:7-44`
- 🟢 **No secrets committed in commerce.** `services/commerce/.env` contains real `JWT_SECRET`/`COOKIE_SECRET`/DB password, but `git check-ignore services/commerce/.env` returns it (ignored) and `.gitignore` lists `.env`. `.env.template` ships empty placeholders. — `services/commerce/.gitignore`, `services/commerce/.env.template`
- 🟢 **`add-certification.ts` is idempotent.** `query.graph` for existing handle → skip before `createProductsWorkflow`. — `services/commerce/src/scripts/add-certification.ts:17-25`
- 🟢 **RLS tests are real, not stubbed.** `test_rls.py:41-94` inserts as tenant A (`db_session(wid_a)`), queries as tenant B (`db_session(wid_b)`), asserts `count is None` / `count == 0`. They are skipped without Postgres (`requires_postgres` mark), not faked. The non-DB `test_problem_json_on_validation_error` runs and passes. — `services/core-api/tests/test_rls.py`, `services/core-api/tests/conftest.py:31-34`

## Risk
- **`/v1/trace/{trace_id}` is a live 500** on every call. Any fleet UI or CLI polling trace by ID is broken. The dead body also encodes the SEC-003 cross-tenant read; an uncareful "just add `def`" fix re-opens it. Severity: high (functional + latent security).
- **Trainer test suite is red.** CI is not green for `services/trainer`. Either the spec 0011 skeleton is shipping without its body, or the tests were merged ahead of the implementation. Severity: medium (signal noise → real failures get ignored).
- **Worker auto-deploy ignores eval gates.** A charter with `modelVersion: "*"` lets a model that failed `eval_harness` gates reach `deploy_model` + `index_collection_for_model`. Spec 0008's deploy gate is effectively advisory in the worker path. Severity: medium (bad model served to a tenant).
- **`document_chunks` missing from ORM metadata + swallowed `ensure_schema` failure.** A first-boot dev path without Alembic produces a broken, RLS-less schema. Severity: medium (dev-only today; production entrypoint runs Alembic, but `docker-entrypoint.sh` continues on migrate failure per SEC-011).
- **Single shared admin key + unvalidated impersonation header.** If the admin key leaks, every workspace is fully read/writable via `x-synth-workspace-id`. Severity: high if leaked, low otherwise (mitigated by production fail-fast on the default).
- **`seed.ts` duplicates on re-run.** Operator re-seeding after a config tweak creates a second region/sales channel/api key + duplicate products. Severity: low (admin-only, fixable by re-pin).

## Recommendation
1. **Restore `governance.get_trace` with RLS.** Add `def get_trace(trace_id: str, workspace_id: uuid.UUID) -> dict`, open `db_session(workspace_id)`, and filter `TraceSpan.workspace_id == workspace_id`. Update `main.py:77-79` to pass `tenant.workspace_id`. Delete the dead block at `governance.py:142-163`.
2. **Reconcile trainer tests with the skeleton.** Either implement `modal_app.py` (`_post_callback`, real `train_asn`/`evaluate`/`compress_and_register` bodies that write to `ARTIFACTS`) or `pytest.skip`/`xfail` the suite with a reference to spec 0011's "runnable skeleton" note. Do not leave 11 red results in CI.
3. **Make eval gates binding for worker auto-deploy.** In `services/worker/main.py:122`, change to `if gates_passed and handoffs.charter_allows_deploy(...)`; keep the charter gate as the authorization and the eval gate as the quality bar. If charter-only is intentional, document it in spec 0008/0012.
4. **Guard `ensure_schema` + add `document_chunks` to metadata or remove the bootstrap path.** Either declare `DocumentChunk` in `app/models.py` (so `create_all` makes it) and add RLS there too, or make `ensure_schema` raise in non-dev and remove the bare `except: pass` at `main.py:30-33`. Production should refuse to serve if Alembic is not at head.
5. **Confine corpus ingestion to `CORPORA_DIR` (SEC-005).** In `data._load_corpus_docs`, reject absolute paths and verify `CORPORA_DIR.resolve() in candidate.resolve().parents` (mirror `runboard/store.py`/`datalab/okf.py`).
6. **Make `seed.ts` idempotent.** Look up `Hub Storefront` sales channel, `United States` region, and each product handle before creating; skip on hit. Match `add-certification.ts`.
7. **Declare `asn-engine` and `eval-harness` in `core-api/pyproject.toml`** (or move to `[project.optional-dependencies].model`); make `services/worker` a workspace member with its own minimal `pyproject.toml`; move trainer's `torch`/`transformers`/`modal` to optional extras.
8. **Validate admin impersonation (SEC-008).** In `auth.py:54-58`, wrap `UUID(wid)` in try/except → 400, verify the workspace exists (one `db_session` lookup), and log impersonation events. Plan per-admin keys + rotation (SEC-004).
9. **Server-side validate `costUsd` in `record_ledger`** (reject negative, cap per-call) or restrict the tenant-facing `POST /v1/ledger` to metadata-only and move cost burn to worker/admin paths.

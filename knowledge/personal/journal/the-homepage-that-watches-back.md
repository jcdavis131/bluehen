jcamd.com started the week as a static resume. It ended as something
I don't have a clean name for: a homepage whose pulse strip shows the
engine's vitals, whose news feed is the fleet's actual commits, whose
arcade embeds the studio's games, and whose knowledge page renders
this very note — positioned in a map where **distance is meaning**,
because every note is embedded with the same MiniLM backbone the
products serve and projected to 2D with nothing invented.

The knowledge system underneath is deliberately boring: Open Knowledge
Format — markdown files with frontmatter and [[wiki-links]], in a
public git repo. Agents write here (the radar drops lever-tagged paper
notes weekly; the capture inbox catches stray ideas; this journal
collects the stories). I write here. The same files, no vendor, no
lock-in. Karpathy's LLM-wiki idea formalized just enough to survive
contact with a multi-agent org.

The test of a personal knowledge system isn't the visualization — it's
whether it still gets curated in month three. This one has an unfair
advantage: the curators don't sleep ([[the-watcher-and-the-watched]]).
